# Lynx 中的溢出与滚动行为

在 Lynx 中，你可以通过使用 `overflow` 属性来控制元件是否显示超出自身显示边界的内容。取值为 `visible` 时会显示超出区域的内容，`hidden` 时会隐藏。

## 溢出与裁切

Lynx 中的 `overflow` 属性与 CSS 规范中的有所不同，当你使用 `overflow` 时，只有 `visible` 和 `hidden` 两种取值。

- `hidden`

  在 CSS 规范中，`clip` 和 `hidden` 都表示裁切超出自身可显示范围的内容，并不允许用户通过拖拽滚动内容。差别是取值为 `hidden` 时可以通过编程的方式滚动内容，如通过
  `scrollLeft` 等滚动相关的值，其仍是一个滚动容器；而取值为 `clip` 时则不是滚动容器。

  Lynx 中不通过 `overflow` 属性来控制元件是否可以滚动。在 Lynx 中当你对非滚动容器（`<view>`）使用 `hidden` 取值时，表现与 CSS 规范中的 `clip` 一致。

- `visible`

  当你想要子元件的内容可以超出父元件的边界显示时，可以通过给父元件设置 `overflow:visible`，来允许内容渲染到边界的外部。

## 滚动

当你想要元素的内容拥有滚动效果，请使用滚动容器元件 [`<scroll-view>`](/api/elements/built-in/scroll-view) 或 [`<list>`](/api/elements/built-in/scroll-view)。

- `<scroll-view>`：使用 [linear](guide/ui/layout/linear-layout.mdx) 布局的单列可滚动元件。
- `<list>`：支持多列的高性能可滚动元件。

Lynx 中的 `overflow` 属性并不存在 `scroll` 取值，非可滚动的元件不会允许用户通过拖拽或是编程的方式滚动内容。当 `overflow` 属性作用于可滚动元件时：

- `hidden`

  内容会在滚动元件的边界处被裁切，允许用户通过拖拽滚动内容，允许通过编程的方式滚动内容。

- `visible`

  内容不会被截断，允许用户通过拖拽滚动内容，允许通过编程的方式滚动内容。可拖拽的区域不会因为 `visible` 取值而变大，仍然是滚动元件的大小。
