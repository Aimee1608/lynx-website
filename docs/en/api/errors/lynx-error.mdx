# LynxError

`LynxError` is the structure used to represent an error,
carrying several properties such as code, level, and error message,
as well as the getters and setters.
Developers can utilize the attributes and methods for troubleshooting.

## Android

### Fields

- `mSubCode: int`: six-digit numbers used to identify and categorize errors, one for each error.
- `mLevel: String`: indicates the severity of the error, could be one of FATAL/ERROR/WARN.
- `mSummaryMessage: String`: briefly describes the cause of the error.
- `mErrorType: int`: indicates the type of the error, could be one of JAVA_ERROR/JS_ERROR/NATIVE_ERROR.
- `mFixSuggestion: String`: suggestions for handling the error. Optional.
- `mCallStack: String`: stack trace of the error. Optional.
- `mRootCause: String`: root cause, usually originating from outside Lynx, such as errors reported by apps. Optional.
- `mCustomInfo: Map<String, Object>`: custom information that may be useful for troubleshooting. Optional.

### Constructors

```java
public LynxError(int errorCode, String errorMessage)

public LynxError(int errorCode, String errorMessage, String fixSuggestion, String level)

public LynxError(int errorCode, String errorMessage, String fixSuggestion, String level, int errorType)
```

### Methods

```java
public int getErrorCode()

public boolean isFatal()
```
